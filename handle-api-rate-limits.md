# Handle API Rate Limits

* **Throttle requests** to stay within allowed RPS and TPM
* **Queue requests** when traffic spikes
* **Retry on 429** using exponential backoff
* **Batch prompts** to reduce API calls
* **Cache responses** for repeated inputs
* **Use async workers** to control execution rate
* **Apply fallback models** when limits are hit
* **Monitor usage** and alert before exhaustion
