# 🟢 Checkpoint

* <mark style="color:$danger;background-color:purple;">**The graph is compiled with a MongoDB checkpointer:**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`langgraph-checkpoint-mongodb`**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**and**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`MongoDBSaver`**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**(from a connection string or**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`MongoClient`**</mark><mark style="color:$danger;background-color:purple;">**), passed to**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`graph.compile(checkpointer=...)`**</mark><mark style="color:$danger;background-color:purple;">**. Each**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`thread_id`**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**(one recruiter session, or one job requisition) stores full**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**`RecruitingState`**</mark><mark style="color:$danger;background-color:purple;">**&#x20;**</mark><mark style="color:$danger;background-color:purple;">**after every step in Mongo collections (checkpoints + writes).**</mark>
* **Local dev** can still use **`MemorySaver`** for speed; **integration / prod** use the same **`MongoDBSaver`** contract so behavior matches.
* After **`human_checkpoint`**, execution **stops at a saved checkpoint**; nothing is lost (pool, cursor, `relax_count`, traces, last page payload).
* **Resume** uses the same `thread_id` plus the user’s choice (`next_10` / `new_search` / `finalize`) so the next `invoke` / `Command` continues from the exact edge you would have taken if the graph had not paused.
* **Ops:** configure **TTL / retention**, **indexes**, and **encryption at rest** — checkpoint payloads can contain **PII** from resumes.

That is how **multi-turn** “show me the next 10” works without re-running from scratch.
